# Comp Insight Studio

一个面向 HR 薪酬宽表场景的智能分析系统：把业务方的一句自然语言问题，编排成一条完整的分析工作流，自动完成数据校验、结构化分析、多维洞察、报告生成、追问补充和历史沉淀。

它不是一个“只会回答问题”的聊天框，而是一套面向真实业务分析场景的智能薪酬分析工作流。

## 这个项目解决什么问题

HRBP、薪酬团队和业务管理层在看薪酬数据时，常见的痛点不是“拿不到数字”，而是：

- 问题描述很模糊，不知道该先看哪个科目、哪个时间窗口、哪些维度
- 宽表数据量大，手工拆分 BU、级别、绩效、人群结构很费时间
- 一次分析往往要来回追问很多轮，最终报告很难沉淀和复用
- 图表、摘要、正式汇报材料通常分散在不同工具里，分析过程断裂

Comp Insight Studio 试图把这些步骤串成一条完整链路：从问题输入，到分析编排，到报告输出，再到追问、修订和历史沉淀。

## 你可以用它做什么

- 看最近 12 个月某个薪酬科目的结构变化和主要驱动因素
- 快速锁定哪些 BU、级别、绩效分位或年龄段对成本贡献最高
- 识别异常抬升群体、高成本群体和结构性差异
- 生成适合管理层阅读的薪酬分析报告和治理建议
- 在当前报告基础上继续追问，而不是每次重新开始

## 核心能力

### 1. 真实数据优先

- 默认不会自动加载 demo 数据
- 正式分析只基于显式激活的数据源
- 没有活动数据源时，系统会明确提示先导入真实 CSV

### 2. 上传即分析

- 支持上传薪酬宽表 CSV
- 自动执行字段校验、模板识别、SQLite 导入和活动数据源切换
- 大文件也支持通过本地路径激活，而不是只能走浏览器上传

### 3. 多阶段分析助手编排

- 先判断当前问题是否具备完整分析口径
- 如果问题不完整，就引导补齐科目、时间、维度和指标
- 口径锁定后，自动进入正式分析链路

### 4. 可追问、可修订、可沉淀

- 支持流式生成分析报告
- 支持基于当前报告继续追问
- 支持把当前报告按新指令修订成不同版本
- 支持历史记录、报告保存和监控扫描

## 智能分析助手如何编排工作

这个项目的重点，不只是“能回答问题”，而是把一次业务问题编排成一条完整的智能分析链路。

### 它如何工作

1. 接收业务方输入的自然语言问题  
   例如：“为什么最近一个月经济补偿金突然抬升？”

2. 判断当前分析口径是否完整  
   如果缺少科目、时间范围、分析维度或指标，系统不会直接给一个模糊答案，而是先引导补齐口径。

3. 锁定本次分析任务  
   一旦问题边界明确，系统会基于当前活动数据源进入正式分析。

4. 执行结构化数据分析  
   通过 SQL 聚合得到 BU 总览、时间区间变化、分维度结构差异等基础结果。

5. 并发生成多维洞察  
   对职能、级别、绩效分位、年龄等次维度进行拆解，提炼异常群体和结构性差异。

6. 补充外部研究与综合判断  
   如果启用了外部研究能力，系统会引入外部信息作为补充参考，而不是只看内部数据。

7. 输出多层次结果  
   包括执行摘要、完整报告正文、图表呈现和管理建议。

8. 支持追问、修订与沉淀  
   报告生成后，用户可以继续追问、修订当前报告，或将结果保存到历史记录中。

这里的重点是：  
系统不是把每次请求都当成一次孤立问答，而是把一次业务问题组织成“补齐口径 -> 分析 -> 输出 -> 追问 -> 沉淀”的完整闭环。

## 系统工作流

```mermaid
flowchart LR
  A["业务问题输入"] --> B{"分析口径完整吗"}
  B -->|否| C["补齐科目 / 时间 / 维度 / 指标"]
  B -->|是| D["锁定分析任务"]
  C --> D
  D --> E["读取活动数据源"]
  E --> F["结构化 SQL 聚合分析"]
  F --> G["多维度并发洞察"]
  G --> H["外部研究补充判断"]
  H --> I["生成摘要 / 正式报告 / 建议"]
  I --> J["继续追问与报告修订"]
  J --> K["保存历史记录与监控"]
```

## 数据分析流程

这个系统可以理解成三层能力叠加：数据处理层、分析层、展示层。

### 数据处理层

- 上传 CSV 或激活本地路径
- 执行字段识别与模板匹配
- 执行格式、编码、关键列、数值列校验
- 导入 SQLite 并切换当前活动数据源

### 分析层

- 生成 BU 总览
- 按职能、级别、绩效分位、年龄等维度拆解
- 识别异常群体、高成本群体和结构性差异
- 输出趋势判断、原因分析和管理建议
- 可选接入外部研究作为补充参考

### 展示层

- 聊天式入口承接自然语言问题
- 流式进度提示用户当前分析到哪一步
- 摘要页帮助快速阅读重点结论
- 完整报告页承载正式输出内容
- 追问抽屉支持基于当前报告继续深入
- 历史记录和监控视图支持结果复盘

## 可视化与交互体验

这个项目的可视化价值不只体现在“有图表”，而是在于把分析过程和分析结果都设计成可阅读、可追问、可沉淀的体验。

- 用图表和结构化报告同时呈现结果，而不是只给一段文字解释

## 快速开始

### 1. 安装依赖

```bash
python3 -m pip install -r requirements.txt
cd frontend && npm install
```

### 2. 配置环境变量

项目支持通过 `.env` 或 shell 环境变量提供配置。

先复制示例文件：

```bash
cp .env.example .env
```

常用配置项：

- `OPENAI_API_KEY`：启用 LLM 分析
- `OPENAI_BASE_URL`：OpenAI 兼容接口地址，可选
- `OPENAI_MODEL`：模型名称，可选，默认 `gpt-4.1-mini`
- `TAVILY_API_KEY`：启用外部研究，可选
- `ACTIVE_DATASET_PATH`：启动时直接激活某份本地 CSV，可选

### 3. 启动后端

```bash
uvicorn app:app --reload
```

打开 [http://127.0.0.1:8000](http://127.0.0.1:8000)。

### 4. 启动前端开发环境

```bash
cd frontend
npm run dev
```

### 5. 健康检查

```bash
curl http://127.0.0.1:8000/health
```

如返回 `{"status":"ok", ...}`，说明后端已正常运行。

## 数据导入方式

### 方式 A：前端直接上传

适合小到中型 CSV。上传后系统会自动完成：

- 字段识别或模板匹配
- CSV 校验
- SQLite 导入
- 活动数据源切换

### 方式 B：启动时激活本地路径

适合大文件或已有稳定数据目录的场景。

```bash
export ACTIVE_DATASET_PATH="/absolute/path/to/your_salary_wide.csv"
uvicorn app:app --reload
```

### 方式 C：启动后调用本地路径激活接口

```bash
curl -X POST http://127.0.0.1:8000/api/data-source/activate-local \
  -H "Content-Type: application/json" \
  -d '{"path":"/你的绝对路径/薪酬宽表.csv"}'
```

## API 概览

- `GET /`：前端入口页
- `POST /api/upload`：上传、校验、导入并激活真实 CSV
- `GET /api/metadata`：返回科目、维度、时间范围和当前活动数据源信息
- `GET /api/data-source`：返回当前活动数据源元信息
- `POST /api/report`：生成 JSON 报告
- `POST /api/report/stream`：生成流式报告
- `POST /api/follow-up`：基于当前报告继续追问
- `POST /api/report/revise`：按指令修订当前报告
- `GET /api/history`：查看历史记录
- `GET /api/monitor`：查看监控扫描结果
- `GET /health`：健康检查
